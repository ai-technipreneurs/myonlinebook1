Chains of characters
#

The goal of this section is to present chains of characters, to use some operations commonly used in text processing.

Chains of characters are not objects as such it is in a Python programme. They are rather sequences of values (like numerical datatypes) which are used to define objects strictly speaking like lists, tuples -that we will explain later-, etc… Like in the case of mathematical operations achievable with numerical data(addition, multiplication, division, etc..), there also exists a package of operations that serve to manipulate chains of characters.

x='bonj'
x[1:5][-3:]

'onj'

0. Defining a string variable #

A chain of character is defined in a classical way by a direct assignment by using the symbol of equality(as discussed on the section related to creation of variables). The only particularity of characters(variables) compared to numerical variables is that their values must ne specified between quotes during the assignment. The three examples below are illustrations.

ch1 = 'Will you be part of this evening meeting ?'
ch2 = '"yes", he answers.'
ch3 = "Good, I will appreciate it."
ch4 = """This sentence is a long chain of characters containing all types of quotations: " ' « » but also more special characters."""

These four examples show different uses of quotations during the definition of a variable as a chain of characters.

  • In the first example (ch1), one uses simple quotes because there is any inconvenient of doing so in view of the specified chain. One could also use double quotes.

  • In the second example (ch2), one uses simple quotes because the specified chain already contains double quotes as value.

  • In the third example (ch3), one uses double quotes because the text contains single quotations mark which in fact are simple quotes.

  • In the fourth example (ch4), one uses triple quotes because the text contains not only simple quotes, single quotations mark and double quotes, but extends to many lines. In these conditions, the use of triple quotes is required.

1. Chains and subchains Indexing (Slicing) #

1.0 Chains and subchains Indexing (Slicing) #

A chain of characters is a sequence of ordored and indexed values. That means we can access each of the elements of the sequence by specifying its index like in a list. We can also extract some portions of a chain by using slices, of generic notation [start=0]:[stop=len][:step=1]. The indexing in python start from 0 : the first element of a list is the element n°0, the 2nd is the n°1, etc. The n elements of a list are thus indexed from 0 to n-1. The examples below are illustrations.

ch="Christelle"
ch[0] # returns 'C'
ch[2] # returns 'r'
ch[-1] # returns 'e', the last element of ch
ch[:6] # returns 'Christ'
ch[6 :] # returns 'elle'
ch[0:10:2] # returns 'Crsel'

ch[:6] # returns 'Christ'
ch[6 :] # returns 'elle'
ch[0:10:2] # returns 'Crsel'

'Crsel'

The elements of a chain of characters are not defined depending on words separated by spaces but rather by the characters that constitute it. Indeed,even though the chaîn was made by a sentence of many words, the indexing will only be made on the basis of characters that form the long chain. Example :

    ch="This is a chain of many words"
    ch[1] # returns 'h'
    ch[4] # returns ' ', the blank space separating 'This' from 'is'
    ch[0:4] # returns 'This'

One can act on a chain (and more generally on a sequence) by using functions (procedural notion) or methods (Object notion). We discuss that lower.

1.2 Addition of chains of characters (concatenation) #

The addition of chains of characters is the juxtaposition of chains of characters the ones after the others in order to form a unique chain of characters. It is about concatenation. To achieve this operation, one uses the symbol + between the names of chains variables. The obtained result is then assigned to a new variable. Examples :

x = "A small step for a man,"
y = "a huge step for humanity"
z = x + y
print(z) # returns 'A small step for a man,a huge step for humanity'

A small step for a man,a huge step for humanity

In the use of the operator of concatenation, one must pay attention when it is about to associate a numerical value to a chain to form a unique chain. Example:

    x = 'This pen costs'
    y = 5
    z = 'euros'

We want to concatenate x, y and z in order to obtain the sentence ‘This pen costs 5 euros’. In this situation, we can not do x+y+z bacause y is a numerical variable. One must first convert this value by using the function str(). Thus, we have

x = 'This pen costs'
y = 5
z = 'euros'
ch = x+str(y)+z
print(ch) # returns 'This pen costs5euros'

This pen costs5euros

One sees that there is no space between costs and 5, and between 5 and euros. We can then modify the operation of concatenation by inserting blank spaces between values. Thus, one gets :

ch = x + ' '+ str(y) + ' '+ z
print(ch) # returns 'This pen costs 5 euros'

This pen costs 5 euros

This result could be also obtained through a direct expression directe like:

ch = 'This pen costs'+ ' '+ '5' + ' '+ 'euros'
print(ch) # returns 'This pen costs 5 euros'

This pen costs 5 euros

1.4 Formatting #

It often happens that we want to recover the value of a variable and integrate it to a chain of characters to form a new chain of character usable for other purposes. Let the variable price be defined as followed

We want to display a message like "Le prix du stylo est de 2 euros".

The goal of the operation of formatting is to display the value of a variable inside a chain of characters. For that, one has three principal methods:

  • the use of the operation of concatenation “+ “

  • the use of the operation of formatting “%” %

  • the use of the function format “{}”.format().

We will only show an example with the method str.format(). The system of formatting allows to control precisely the conversion of variables into chains of characters. It is essentially based on the method str.format():

print("{name} has {age} years".format(name='Calvin', age=6)) #'Calvin has 6 years'

print("{} has {} ans".format('Calvin', 6)) # Shortcut  'Calvin has 6 years'

Remark: One must do an important remark concerning the difference between a value of character type and a value of alphabetical type. Indeed, an alphabetical value is always a chain of characters whereas a chain of characters is not necessarily an alphabetical value. In python, a character value is always expressed between quotes(simple, double or triple) whereas a numerical value is expressed without quotes. One distinguishes two principal types of data : numerical data and data in characters. Please note that a numerical value expressed between quotes is automatically treated as a character value even if it is not an alphabetical value. It is important to keep in mind these details during the use of sequences of values in python (see example below):

    x = 12 # x is a numerical value
    y = "12" # y is a chain of character formed by two digits
    z = "my text" # z is a chain of character formed by alphabetical values
    k = "This pen costs 5 euros" # k is a chain of character formed by alphanumerical value

For more details on chains of characters see this page.

def nGrams(n, s):
    sn=[s[i:n-1+i] for i in range(0, len(s))]
    return sn
nGrams(3, "Good job")

def sim_str(s,r):
   l_s=nGrams(3, s)
   l_r=nGrams(3, r)
   intersection = [l for l in l_r if l in l_s]
   union=[l for l in l_r or l in l_s]
   sim=len(intersection)/len(union)
   return sim
sim_str("Goog job", "job well done")
